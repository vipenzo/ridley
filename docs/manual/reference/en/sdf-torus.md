# sdf-torus

## Signature

`(sdf-torus R r)`

## Description

Construct an SDF node for a torus centered on the current turtle pose,
with its axis along the turtle's *up* vector. `R` is the major radius
(centre of the tube to the torus axis); `r` is the minor radius (the
tube's own radius). Returns a lightweight SDF tree; no geometry is
computed until meshing.

`sdf-torus` is a convenience over `sdf-formula` plus annotated bounds —
internally it expands to a quoted radial distance expression with the
ring shape baked in. The auto-bounds carry the right axis-aligned box,
so meshing resolution scales correctly without manual hints.

> Desktop only: requires the libfive backend.

## Parameters

- `R` — major radius (distance from the centre of the tube to the
  torus axis).
- `r` — minor radius (tube radius). Must be less than `R` for a
  classic ring shape; `r ≥ R` produces a self-intersecting "spindle"
  torus.

## Notes

- The axis tracks the turtle's *up* vector at construction time. To
  re-orient an existing torus, use `rotate` (e.g. `(rotate ring :x 90)`
  rotates it onto the XZ plane).
- Mesh smoothness around the tube is driven by the turtle resolution
  at construction time. The default (`:n 64`) gives ~64 voxels around
  the minor circumference. For a smoother torus, increase resolution
  in scope: `(turtle (resolution :n 256) (sdf-torus 16 6))`. The same
  knob controls mesh primitives (`cyl`, `cone`, `sphere`), so a single
  setting affects both worlds consistently.
- For a torus generated by revolving a custom 2D profile (any shape,
  not just a circle), use `sdf-revolve` with a 2D SDF profile.
